SQL Injection Prevention

In order to prevent SQL Injections, the user input needs to be sanatized in order to make sure that users are not able to input malicious code.

How can I Prevent SQL Injection?

  • Do not allow Dynamic SQL. By abstaining the use of Dynamic Sql and only using prepared statements it makes sure that only preallocated and authorized commands can be sent through the system. This technique should be used with formats of security in order to maximize security.
  • Sanitize user input, for example Phone Number fields should only allow numbers found in phone numbers, and e-mail addresses should only allow the characters available in an e-mail address.
  • Use the LIMIT Sql control within queries to bypass mass disclosure of records in the event of an SQL Injection.
  • Web Application Firewall. These can be used in order to help filter malicious code that can be sent to the application. A good example of a WAF is ModSecurity.
  • Encryption or Hashing. Protecting all passwords or data that needs to be kept secret in order to make sure that they are not displayed as clear text is an important factor of security. Recently Facebook was found to be storing hundreds of millions of passwords in clear text.
  • Monitoring. Simply monitoring the SQL statements that are coming from database-connected applications will help to make sure that the application is kept safe. Any and all security flaws should be acted upon immediately in order to stop the wide spread damage that could occur from these different kinds of attacks.

One important thing to remember is that security breaches over time will become more and more sophisticated, and it's important to keep yourself and your business up to date with security.

Remember

Understanding how these attacks are composed allow for you to gain a better insight as to how you can protect yourself, and others from these types of attacks.

The OWASP foundation rated SQL Injection as their number one exploit in their 2013 report, and their latest 2017 report.
