How to Create a Strong Password.

How can I make one?

There are several different ways to create strong passwords:

Passphrases:
Since you may not know when these attacks are happening, the best preventative measure is to be cautious, don’t use unsecured public Wi-Fi connections, don’t purchase goods on a mobile connection. These connections can be hijacked - only use Wireless connections that you trust such as a protected Home network, or a protected Private network.
Password Managers:
Password Managers act as a middleman for remembering Passwords and often will have features to create strong passwords from the application. Some services such as Dashlane even have notifications when a website has been breached and will prompt you to change your password for that website. Typically, these services will have applications for all devices to make sure that no password is forgotten and can even allow for one click logins that will auto fill your information into the required fields.
Random Strings:
The most secure password is the hardest to remember. An example of this would be as follows: 't!1Df3&Zm,'.

What makes a Weak password?

Dictionary Words:
Words that appear in the dictionary without any modification are the easiest to crack, for example ‘exclusive’. Typically when trying to breach passwords there are two formats that will be followed and these are Dictionary Attacks, and Brute forcing. Using a list of words from a dictionary will mean that common words are vulnerable to attack from a Dictionary Attack.
Names:
Names are a common password that are used i.e. ‘Kenan’ are used a lot. The same applies as previously, adding singular numbers are a weak attempt at trying to improve password strength, as different variations of these passwords will be within the attack words list, for example it could feature variations such as ‘Kenan1’, ‘Kenan11’, or ‘Kenan1995’.
Random Strings:
The most secure password, is the hardest to remember. An example of this would be as follows: 't!1Df3&Zm,'. Common keystrokes are to be avoided, an example of one of these would be ‘1qaz2wsx’. Whilst in theory this does have some of the characteristics that would make for a good password, it is also using something very common – the letters from two lines from a QWERTY keyboard, one of the most common types in the world.

Avoid using Password Strength Checkers

Entering a password on an unknown system to test it's strength reduces it's integrity. Potentially adding it to a list of passwords to be used to breach systems.
